# XiHan.Framework.Bootstrap

## 项目定位

`XiHan.Framework.Bootstrap` 用于承载框架引导层默认实现。

它不是最小核心，也不是业务层能力，而是“宿主如何把核心、契约和可选模块运行时真正装配起来”的实现层。

## 责任边界

本项目负责：

1. 模块发现相关默认实现
2. 程序集发现与类型发现默认实现
3. 模块描述、依赖排序和生命周期编排默认实现
4. 应用工厂与应用对象默认实现
5. `IServiceCollection` 接入扩展

本项目不负责：

1. 最小核心原语
2. 业务领域建模
3. 基础设施 Provider 实现
4. 最终宿主入口代码

## 依赖方向

本项目依赖 `XiHan.Framework.Kernel` 与 `XiHan.Framework.Bootstrap.Abstractions`。

宿主项目、模块运行时项目或需要扩展引导过程的上层能力，应依赖本项目或其抽象，而不是反向向 `Kernel` 塞入启动逻辑。

## 当前阶段

当前项目先承接从 `Kernel` 拆出的反射发现默认实现。

当前已具备的最小引导链路包括：

1. `DependsOn` 模块依赖解析
2. 模块描述与依赖排序
3. 模块初始化与关闭编排
4. 内部/外部服务提供器两种应用对象
5. 应用工厂与 `IServiceCollection` 扩展

后续将继续承接更完整的应用工厂、模块描述模型扩展、模块图构建、宿主集成与运行时编排等偏引导层职责。
