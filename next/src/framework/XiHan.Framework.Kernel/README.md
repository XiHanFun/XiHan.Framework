# XiHan.Framework.Kernel

## 项目定位

`XiHan.Framework.Kernel` 是整个 `XiHan.Framework.Next` 的最小内核项目。

它只负责框架得以建立和运行所必需的最基础机制，不承载业务模型、不承载基础设施实现、不承载大而全的工具能力。

## 责任边界

本项目负责：

1. 模块最小抽象
2. 生命周期最小抽象
3. 框架级基础异常
4. 极少量全局稳定约定

本项目不负责：

1. 分页、响应模型、DTO 等共享契约
2. 领域实体、聚合、仓储
3. 日志、缓存、数据库、消息等基础设施实现
4. 本地化、验证、安全等可独立演进能力

## 依赖方向

原则上本项目应尽量保持零项目依赖，作为其他框架项目的最底层依赖。

## 设计约束

1. 保持极薄
2. 保持稳定
3. 不接受“暂时没地方放”的代码
4. 不引入与具体技术实现绑定的依赖

## 当前阶段

当前项目已建立最小骨架和基础说明，后续将继续补充模块生命周期、模块依赖声明与启动装配相关能力。
