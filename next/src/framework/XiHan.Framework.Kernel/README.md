# XiHan.Framework.Kernel

## 项目定位

`XiHan.Framework.Kernel` 是整个 `XiHan.Framework.Next` 的最小内核项目。

它只负责框架得以建立和演进所必需、且十年后大概率仍然稳定的最小原语。

本项目明确不再承担“框架引导器”“运行时装配器”“反射扫描中心”这类职责，以避免重演旧 `Core` 大一统膨胀问题。

## 责任边界

本项目负责：

1. 最小模块契约
2. 对象访问器与类型集合等稳定原语
3. 框架级基础异常
4. 极少量全局稳定约定

本项目不负责：

1. 应用创建、启动、关闭等宿主抽象
2. 模块发现、程序集扫描、类型查找等运行时装配能力
3. 分页、响应模型、DTO 等共享契约
4. 领域实体、聚合、仓储
5. 日志、缓存、数据库、消息等基础设施实现
6. 本地化、验证、安全等可独立演进能力

## 依赖方向

原则上本项目应尽量保持零项目依赖，作为其他框架项目的最底层依赖。

## 设计约束

1. 保持极薄
2. 保持稳定
3. 不接受“暂时没地方放”的代码
4. 不引入与具体技术实现绑定的依赖
5. 不承担宿主与运行时装配职责

## 当前阶段

当前项目已完成第一轮收边界：

1. 保留最小模块契约和稳定原语
2. 将应用引导与生命周期抽象迁移到 `XiHan.Framework.Bootstrap.Abstractions`
3. 将反射发现与运行时装配迁移到 `XiHan.Framework.Bootstrap`

后续本项目只继续吸收真正属于最小核心的能力，不再继续迁入旧 `Core` 中偏运行时、偏宿主、偏扫描机制的实现。
