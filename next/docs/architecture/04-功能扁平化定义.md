# XiHan Framework — 功能扁平化定义

> 版本：1.0.0-preview.1 | 状态：草案 | 最后更新：2026-05-09

---

## 零、设计原则

**每个功能独立声明，不存在层级归属。** 功能之间通过能力图谱（Capability Graph）声明依赖关系，而非通过命名空间层级隐含从属。

一个功能 = 一个独立的 NuGet 包。使用者按需引用，按需 `UseXxx()`。

## 一、V1 内核功能（8 项 — 必须存在）

### F001 — 核心抽象
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel` |
| **能力标识** | `kernel:core` |
| **稳定性** | Stable |
| **依赖** | 无（仅 `Microsoft.Extensions.*`） |
| **职责** | 所有框架核心接口、契约、基础类型 |

**包含类型**：
- **[冻结]** `Result<T>`, `Error` — 结果模型，永不破坏
- **[冻结]** `Maybe<T>` — 可选值，类型安全地表达"可能没有值"
- **[冻结]** `TypedId<T>` — 强类型 ID，防止原始类型误赋值
- `XiHanMetadata` — 框架身份元数据
- `IXiHanFeature` — 特性契约
- `XiHanException` — 基础异常
- `ApiLevelAttribute`, `Stability` 枚举 — API 稳定性标记

> **[冻结]** 标记的类型属于框架的 L0 原语层。即使 MAJOR 版本升级，也不允许破坏性变更。这是框架最底层的"用户态 ABI"。

**不做**：
- 不包含时间抽象（.NET 原生 `TimeProvider` 已足够）
- 不包含 ID 生成（进 V2 `XiHan.Framework.DistributedIds`）
- 不包含实体契约（进 V2 `XiHan.Framework.Domain.Abstractions`）
- 不包含事件总线（进 V2 `XiHan.Framework.EventBus`，需要 DI 依赖）
- 不包含任何具体实现、基类
- 不依赖第三方库（除 `Microsoft.Extensions.*`）

### F002 — 应用宿主
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel.Hosting` |
| **能力标识** | `kernel:hosting` |
| **稳定性** | Stable |
| **依赖** | `XiHan.Framework.Kernel (>=1.0.0)` |
| **职责** | 应用构建、特性装配、启动/停止生命周期 |

**包含类型**：
- `XiHanAppBuilder` — 应用构建器。支持包装外部 `IServiceCollection` + `IConfiguration`（宿主集成）或独立创建
- `XiHanApp` — 应用实例
- `IHostedFeature` — 需要 Start/Stop 生命周期的特性接口
- `XiHanFeatureExtensions` — 特性注册与 `IHostBuilder` 集成扩展
- `HostedFeatureWrapper` — 将 `IHostedFeature` 桥接为 `BackgroundService`

**API 示例**：
```csharp
// 独立模式
var app = XiHanApp.CreateBuilder(args)
    .UseFeature<MyFeature>()
    .Build();

// 宿主集成模式（共享 DI 容器）
builder.Host.UseXiHan(xihan => xihan.UseFeature<MyFeature>());
```

**不做**：
- 不定义模块生命周期钩子
- 不处理 HTTP 请求（那是 `XiHan.Framework.Hosting.AspNetCore` 的职责）

### F003 — 能力图谱
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel.Capability` |
| **能力标识** | `kernel:capability` |
| **稳定性** | Preview |
| **依赖** | `XiHan.Framework.Kernel (>=1.0.0)` |
| **职责** | 能力声明、发现、协商、依赖校验、版本约束 |

**包含类型**：
- `ICapability` — 每个扩展包实现此接口声明自己的能力
- `CapabilityDescriptor` — 能力描述符（名称、提供的能力、依赖的能力、版本约束）
- `CapabilityGraph` — 能力依赖图（拓扑排序、循环检测、缺失检测、版本约束）
- `CapabilityValidationResult` — 验证结果模型（缺失、循环、版本不匹配）
- `CircularDependencyException` / `MissingCapabilityException` — 分类异常

**能力清单格式**（每个包根目录 `xihan.manifest.json`）：
```json
{
  "name": "XiHan.Framework.Security.Authorization",
  "version": "1.0.0",
  "stability": "stable",
  "provides": ["auth:rbac", "auth:abac"],
  "requires": {
    "XiHan.Framework.Kernel": ">=1.0.0 <2.0.0",
    "XiHan.Framework.Security.MultiTenancy": ">=1.0.0 <2.0.0 | none"
  },
  "configSchema": "config/authorization.schema.json"
}
```

**不做**：
- 不实现 IoC 容器（使用 MS DI）
- 不管理运行时模块加载/卸载（V1 仅编译时）

### F004 — 执行管道
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel.Pipeline` |
| **能力标识** | `kernel:pipeline` |
| **稳定性** | Stable |
| **依赖** | `XiHan.Framework.Kernel (>=1.0.0)` |
| **职责** | 统一请求执行管线、中间件组装、管道上下文 |

**包含类型**：
- `IPipelineMiddleware` — 中间件接口
- `PipelineBuilder` — 管道构建器。支持 `Use`/`UseAt`/`UseBefore`/`UseAfter`，通过 `_middlewareTypes` 元数据精确查找中间件
- `PipelineContext` — 管道执行上下文（携带 User, Tenant, TraceId，不假设 HTTP）
- `PipelineHandler` — 管道处理器委托 `Task Invoke(PipelineContext ctx)`
- `IStartupFilter` — 模块控制中间件插入位置
- `PipelinePosition` — 中间件位置枚举（Beginning/End/Before/After）

**API 示例**：
```csharp
var pipeline = new PipelineBuilder()
    .Use<UnitOfWorkMiddleware>()
    .Use<ValidationMiddleware>()
    .Use<AuthorizationMiddleware>()
    .Use<CachingMiddleware>()
    .Build();

var result = await pipeline.ExecuteAsync(context, handler);
```

**`IStartupFilter` 示例**：
```csharp
// Feature 可通过实现 IStartupFilter 声明中间件位置
public class AuthorizationFeature : IXiHanFeature, IStartupFilter
{
    public Action<PipelineBuilder> Configure(Action<PipelineBuilder> next)
    {
        return builder =>
        {
            // 鉴权中间件插到管道最前面
            builder.UseAt<AuthorizationMiddleware>(PipelinePosition.Beginning);
            next(builder);
        };
    }
}
```

**不做**：
- 不绑定 HTTP 请求——`PipelineContext` 不包含 `HttpContext`，通用管道可用于消息、后台任务等
- 不强制中间件顺序——`IStartupFilter` 是可选机制，用户可完全显式排列

### F005 — 可观测性
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel.Diagnostic` |
| **能力标识** | `kernel:diagnostic` |
| **稳定性** | Preview |
| **依赖** | `XiHan.Framework.Kernel (>=1.0.0)` |
| **职责** | 日志、指标、追踪、审计事件的抽象层和轻量实现 |

**包含类型**：
- `IDiagnosticLogger` — 结构化日志接口
- `IDiagnosticMeter` — 指标记录接口（计数器、仪表、直方图）
- `IDiagnosticTracer` — 分布式追踪接口
- `IAuditSink` — 审计事件接收器接口
- `DiagnosticLevel` — 日志级别枚举（Trace/Debug/Information/Warning/Error/Critical）
- `AuditEvent` — 审计事件模型

**约定**：
- 框架自带轻量实现开箱即用，满足 99% 日常场景
- 所有能力包通过此包报告遥测数据
- 生产者-消费者模式：能力包生产数据，通过 DI 注入的 Sink 消费
- 提供 `XiHan.Framework.Diagnostic.OpenTelemetry` 桥接包（V2），供需要对接外部 APM 的用户使用

**不做**：
- 不绑定 OpenTelemetry 或其他 APM 厂商
- 不实现持久化存储（由消费者决定）

- 不包含用户存储

### F007 — 插件系统
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Kernel.Plugins` |
| **能力标识** | `kernel:plugins` |
| **稳定性** | Preview |
| **依赖** | `XiHan.Framework.Kernel (>=1.0.0)` |
| **职责** | 插件加载、隔离、通信、生命周期管理 |

**已实现**：
- `IPlugin` — 插件契约，声明名称、版本、依赖
- `IPluginHost` — 插件宿主接口
- `IPluginContext` — 插件隔离上下文接口
- `StaticPluginLoader` — 编译时插件加载器。要求插件有 `new()` 约束，使用 `Activator.CreateInstance`。

**计划中（V2）**：
- `DynamicPluginLoader` — 运行时 `AssemblyLoadContext` 隔离加载
- 源生成器消除 `Activator.CreateInstance`（完整 AOT 兼容）
- `UsePlugin<T>()` Builder 扩展

**AOT 兼容性**：
- 当前版本部分 AOT 兼容（插件必须有 `new()` 约束，内部使用反射创建）
- 完整 AOT 兼容需 V2 源生成器版本
- 插件间通信仅通过接口，不直接引用类型

**不做**：
- 不实现热重载（V2+）
- 不实现插件市场/远程下载

### F008 — ASP.NET Core 适配
| 属性 | 值 |
|------|-----|
| **包名** | `XiHan.Framework.Hosting.AspNetCore` |
| **能力标识** | `hosting:aspnetcore` |
| **稳定性** | Stable |
| **依赖** | `XiHan.Framework.Kernel.Hosting (>=1.0.0)` |
| **职责** | 框架与 ASP.NET Core 的胶水层 |

**已实现**：
- `XiHanWebApplicationExtensions` — `AddXiHan()` / `UseXiHan()` 扩展方法，共享宿主 DI 容器

**计划中（V2）**：
- `XiHanWebHost` — Web 宿主
- `XiHanMiddleware` — 将 Pipeline 注册为 ASP.NET 中间件
- 端点路由集成

**API 示例**：
```csharp
var builder = WebApplication.CreateBuilder(args);

builder.AddXiHan(xihan =>
{
    xihan.UseFeature<AuthorizationFeature>();
    xihan.UsePipeline(pipeline =>
    {
        pipeline.Use<TenantResolutionMiddleware>();
        pipeline.Use<AuthorizationMiddleware>();
    });
});

var app = builder.Build();
app.UseXiHan();
app.Run();
```
