# XiHan Framework — 开发计划

> 版本：1.0.0-preview.1 | 状态：草案 | 最后更新：2026-05-09

---

## 一、总览

### 1.1 版本路线

```
2026 Q3      V1.0-preview.1   内核 7 包 + 2 示例特性包
2026 Q4      V1.0-preview.2   管道完善 + 能力图谱 +
                               源生成器原型 + 测试覆盖
2027 Q1      V1.0-rc.1        API 冻结 + 性能基准
2027 Q2      V1.0.0           正式发布
2027 Q3-Q4   V1.x             稳定修复 + 扩展包孵化
2028+        V2.0             扩展包正式发布 + BasicApp 重建
```

### 1.2 开发原则

| 原则 | 说明 |
|------|------|
| **测试先行** | 每个公共 API 先写测试，再写实现 |
| **文档同步** | API 变更同步更新文档，不滞后 |
| **语义版本** | 严格遵守 SemVer 2.0 |
| **API 分级** | 所有公共类型必须标记 Stability 级别 |
| **PR 最小化** | 单 PR 不超过 500 行变更 |
| **持续集成** | 每次 push 运行完整构建 + 测试 + 分析器 |

## 十、V2+ 长期规划

### 10.1 BasicApp 重建

新框架 1.0 稳定后，用新框架重建 `XiHan.BasicApp`：
- 保留旧 BasicApp 的业务场景（Saas 管理、组织架构、权限管理）
- 使用新框架的特性组合模式重写
- 作为新用户的官方模板

## 十三、风险管理

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| API 设计反复 | 中 | 高 | Phase 1 前完成 API 设计审查并冻结 |
